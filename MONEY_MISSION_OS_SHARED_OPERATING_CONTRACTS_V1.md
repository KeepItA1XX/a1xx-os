# Money Mission OS — Shared Operating Contracts v1

## Canonical record minimum

Every project, client, job, skill, worker, output, approval, task, and export record must expose: stable ID, source, freshness, owner, status, confidence, next action, destination, approval state, related records, and receipt ID.

## Domain workspace minimum

Each primary agent workspace contains: role charter, current priorities, authoritative sources, KPIs, active work, staff map, tool health, permissions, decisions, assumptions, risks, inbound queue, outbound handoffs, daily brief, closeout, evaluation score, and version history.

## Handoff minimum

Every handoff includes sender, recipient, purpose, source records, requested action, owner, deadline, destination, approval state, confidence, blockers, and completion evidence.

## Tool/API health minimum

Every integration exposes authentication state, reachability, allowed actions, read/write boundary, freshness, retry policy, failure message, test fixture, and destination mapping.

## Receipt minimum

Every staged or executed action records action ID, actor, authority level, inputs, proposed action, result, destination, timestamp, approval reference, and verification status.

## Learning and promotion

Profile or workflow changes require a version, reason, affected scenarios, golden-set evaluation, regression result, approval owner, and rollback reference. Agents may not change their own authority, identity, or approval policy.
